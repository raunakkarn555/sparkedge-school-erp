📋 Complete Workflow Guide
Everything you need to know — from adding your first teacher to exporting and syncing data across devices. Read this once and you'll be fully independent.
Who Can Access This Page?
Owner
Full access — can add, edit, delete teachers, toggle status, export/import data, and see salary details.
Staff (with Teachers permission)
Can view and add teachers. Cannot see salary. Cannot delete unless owner grants full access.
🔑 Login: Open teachers.html → enter your password (same password you use for admin.html). The system reads passwords from school_data.js. No separate teacher-portal password — one password per person for all portals.
How Passwords Work
Fully contained in school_data.js — no database, no server
1
All passwords live in school_data.js (in the same folder as this file). There is no database or server — everything is hardcoded.
2
Owner password → OWNER_PASSWORD = 'Raunak@123' (line ~28 of school_data.js). Change this value and push to update.
3
Staff passwords → each member's password field in the STAFF_MEMBERS = [...] array. Each person has their own unique password.
4
To add a staff member: Go to admin.html → Team Members → Add Member → fill form → Download Config → paste the code block into STAFF_MEMBERS in school_data.js → git push.
✓
After git push, GitHub Pages rebuilds in ~60 seconds. The new member can log in immediately.
⚠️ Important: Passwords are visible in the JS file to anyone with access to your GitHub repo. Set repo to Private or use GitHub Pages with a protected branch if you need confidentiality.
How to Add a Teacher
1
Click + Add Teacher button (top-right of this page). A large form modal opens.
2
Fill in Basic Info: Full name, Employee Code, Designation (e.g. PGT, TGT, PRT), Department, Date of Birth, Date of Joining.
3
Fill in Contact: Phone, Email, Address.
4
Set Subjects Taught (comma-separated, e.g. Mathematics, Physics) and optionally assign a Class & Section and mark as Class Teacher.
5
Optional: Upload Photo (click the photo box, select a JPG/PNG — stored as base64 in localStorage).
6
Click Save Teacher. The record is saved immediately to your browser's localStorage. You'll see it appear in the grid.
✓
The teacher ID is auto-generated as TCH-YYYY-NNN (e.g. TCH-2025-001). Never change it after creation — it links to timetable and other records.
Editing, Deactivating & Deleting
✏
Edit: Click the pencil icon on any teacher card. The same form opens with all current data pre-filled. Make changes → Save.
⏸
Deactivate (recommended over delete): Click the pause icon → teacher is marked inactive and greyed out. Their records (timetable, assignments) are preserved. Reactivate any time with the play button.
🗑
Delete (permanent): Click the trash icon → confirmation dialog → teacher removed. Warning: This is irreversible. Prefer Deactivate unless the record was entered by mistake.
Export & Import — How Data Syncs Across Devices
💡 Why is this needed? Since there's no server, data lives in each browser's localStorage. Export/Import is how you move data from one device to another — e.g. Owner adds teachers on laptop → shares JSON → colleague can see them on tablet.
📤
Export: Click Export Data (sidebar or topbar) → downloads school-data-YYYY-MM-DD.json. This contains all teachers, students, fees, marks, announcements etc.
📤
Share the JSON: Send via WhatsApp, Google Drive, Email, or USB. The JSON contains no passwords — only school data.
📥
Import (recipient): Open teachers.html or admin.html → click Import Data → Upload .json file or paste content → click Import & Merge.
✓
Merge logic: Records with matching IDs are updated. New records are added. Nothing is deleted. It is always safe to import.
✓
Best practice: Owner exports after every session and shares the file with the team. Team imports at the start of each session. This keeps everyone in sync.
⚠️ Conflict rule: If two people edit the same teacher record and then import, the last import wins. Avoid simultaneous edits to the same record.
For Teachers (Staff Login)
1
Your login password is given to you by the Owner. Open teachers.html and type it in the password box.
2
You can only see sections the Owner has granted permission for. If you try to access a locked section, you'll see a "Permission Denied" notice — contact your admin.
3
To update your own profile info (phone, address, etc.), ask the admin. Staff cannot edit teacher records unless they have the teachers permission enabled.
✓
The session expires when you close the browser tab. Next time you open the page, you'll need to log in again.
