🧑‍💼 Staff Team Add Karo — school_data.js Edit

⚠️ File edit required 👑 Owner only
⚠️ Kyun file edit karni padti hai?
Passwords security ke liye code mein hardcoded hain — localStorage mein nahi. Isse koi browser tools se password hack nahi kar sakta. Par iska matlab hai ki har naya staff member add karne ke liye school_data.js file kholo aur STAFF_MEMBERS array mein block add karo.
1
school_data.js file text editor mein kholo
Notepad, VS Code, ya koi bhi text editor chalega. File saari HTML files ke saath same folder mein hai.
2
STAFF_MEMBERS array dhundho (line ~43)
Yahaan ek example block already commented hai. Ek naya block add karo isi format mein:
// school_data.js → STAFF_MEMBERS array mein add karo: { id: 'STF-2026-002', // Unique ID — kabhi change mat karo name: 'Priya Sharma', // Display name role: 'Accountant', // Sirf display ke liye password: 'Priya@2025', // ← Unique password — share karo WhatsApp pe active: true, permissions: { teachers: false, // Teachers portal dekh sakta hai? students: true, // Students portal fees: true, // Fee management marks: false, // Marks portal docs: true, // Certificates timetable: false, // Timetable announcements: true, // Announcements export: true, // Export/Import data settings: false, // School settings (sirf owner ke liye true) } },
3
File save karo → GitHub push karo (agar GitHub Pages use kar rahe ho)
git add . → git commit -m "Add staff member" → git push
GitHub Pages ~60 seconds mein update ho jaata hai. Fir staff member apne password se login kar sakta hai.
4
Staff ko disable karna ho toh?
Block delete mat karo — sirf active: false kar do. ID preserve rahega, data safe rahega, login band ho jaayega.

💾 Data Management
Export & Import — Sabse Zaroori Skill
Yeh ERP ka backbone hai. Samajh lo — warna data loss hoga.
🚨 Yeh system NO SERVER hai

Saara data sirf aapke browser ke localStorage mein save hota hai — koi cloud nahi, koi server nahi, koi automatic sync nahi. Iska matlab: agar browser data clear ho gaya, ya aap dusra device use karo, toh data nahi milega.

Export = Backup = Survival. Har important kaam ke baad Export karo.

Export kaise karein — Step by Step
1
Kisi bhi portal mein login karo (admin.html preferred)
Owner ya authorized staff login karo.
2
Sidebar ya topbar mein "Export" button dhundho aur click karo
Ek JSON file automatically download ho jaayegi — naam hoga: school-data-2025-06-15.json (aaj ki date ke saath).
3
Yeh JSON file safe jagah save karo — multiple copies
• Google Drive — "School ERP Backups" folder banao
• WhatsApp — Apne aap ko ya owner ke group mein bhejo
• Pen Drive / Hard Drive — Physical backup
• Email — Apne aap ko email karo
Import kaise karein — Step by Step
1
Naye device ya browser pe portal open karo → Login karo
Fresh browser mein data khali hoga — sirf login karoge toh kuch data nahi dikhega.
2
Sidebar mein "Import" button click karo
Import modal khulega. Yahaan do options hain:
3
Option A — JSON file upload karo
"📂 Upload .json file" button click karo → Apni backup file select karo → "Import & Merge" click karo.
4
Option B — JSON paste karo
Kisi ne WhatsApp pe file bheja par attachment open nahi ho raha? File ka content copy karo → Text box mein paste karo → "Import & Merge".
5
Page reload hoga → Poora data wapas aa jaayega
✅ "Imported & merged!" toast dikhega. Ab sab kuch local storage mein hai. Kaam shuru karo.
✅ Merge kaise kaam karta hai
Import "overwrite" nahi karta — merge karta hai. Matlab agar aapke paas purana data tha aur naya JSON import kiya, toh naye records add ho jaayenge aur matching IDs update ho jaayenge. Koi record delete nahi hoga import se. Par: matching ID wale records overwrite ho jaayenge — toh sahi latest file import karo.

Export file ek hi hoti hai — school-data-[date].json — ismein saara data hota hai (students, teachers, fees, marks, timetable — sab). Isliye ek export se full backup ho jaata hai. Alag-alag portal ka alag backup nahi hota — sab ek JSON mein hain.
🚨 Critical Risk
LocalStorage — Khatron Ki Poori List
LocalStorage ek temporary browser storage hai. Yeh situations mein data WIPE ho sakta hai:
🚨 Data Delete Hone Ki Situations

Neeche di gayi har situation mein aapka poora school data bina warning ke wipe ho sakta hai:

🧹 Browser Cache Clear

Chrome/Firefox mein "Clear browsing data" kiya → "Cookies and site data" include tha → Data gone. Bahut common galti hai.

🔒 Incognito / Private Mode

Incognito window mein kaam kiya → Tab band kiya → Data gone. Incognito mein localStorage session end pe delete hota hai.

💻 Different Device

Ghar ka laptop → School ka computer — alag devices hain, alag localStorage. Data share nahi hoga automatically.

🔄 Different Browser

Chrome mein data tha, Firefox khola → Data nahi milega. Har browser ki apni alag localStorage hoti hai.

📱 Phone Storage Full

Phone ki storage full hone pe browser purani localStorage entries delete kar sakta hai apni marzi se.

🔧 Browser Reinstall / Update

Chrome uninstall/reinstall kiya ya major update → LocalStorage wipe ho sakti hai.

💥 System Crash / Format

PC format ho gaya → Browser data gone → School data gone. Koi cloud backup nahi tha toh kuch nahi milega.

⏰ Storage Quota Exceed

Bahut zyada photos upload kiye (base64) → localStorage 5-10MB limit exceed → Browser error ya data corrupt.

🛡️ Prevention
Backup Routine — Isko Apni Aadat Banao
✅ Recommended Backup Schedule

Daily: Agar aaj koi bada kaam kiya (marks enter kiye, fees record kiye, students add kiye) → aaj hi export karo → Google Drive pe upload karo.

Weekly: Har Friday school band hone se pehle → Export karo → WhatsApp group mein bhejo (backup copy ke liye).

Monthly: Month ke aakhir mein → Export → Pen drive pe bhi copy rakho.

Session start/end pe: Har academic session shuru aur khatam hone pe → Export karo → Ek permanent archive rakho (rename karke: school-data-2025-26-FINAL.json).

📁 Google Drive

Best option. "School ERP" folder banao → andar "Backups" folder → Har JSON file date ke naam se upload karo. Free 15GB storage milti hai.

💬 WhatsApp

Owner ek personal WhatsApp group banaye sirf apne liye (ya trusted person ke saath). Har export wahan forward karo. Easy aur fast.

🖥️ Pen Drive

Physical backup. School office mein ek designated pen drive rakho sirf ERP backup ke liye. Monthly copy karo wahan.

🚨 Photos ke baare mein
Student aur teacher photos base64 format mein localStorage mein store hoti hain — yeh bahut space lete hain. Agar bohot saare photos upload kiye toh localStorage 5MB limit pe aa sakta hai. Solution: Photos upload karna optional rakho — sirf zaroorat pe. Ya photos chhoti rakhne ke liye compress karke upload karo.

💡 Pro Tips
Best Practices & Important Warnings

✅ Ek Browser Fix Karo

School ka kaam sirf ek hi browser mein karo — Chrome ya Firefox — consistent raho. Dono mein alag localStorage hoti hai.

✅ Export Shortcut Yaad Rakho

Roz kaam khatam karne se pehle Export → Drive upload. 30 second ka kaam hai. Ek baar ki mehnat vs kabhi data loss nahi.

⚠️ Incognito Kabhi Use Mat Karo

Tab close hone pe saara data wipe. Normal window use karo hamesha jab ERP pe kaam karo.

⚠️ Photos Chhoti Rakho

Student photos upload karne se pehle compress karo (TinyPNG.com pe free compression milti hai). 5MB localStorage limit jaldi bhar jaati hai photos se.

💡 Roll Number Unique Rakho

Ek class mein roll number repeat mat karo. System automatically check nahi karta — aapko manually dhyan rakhna hoga.

💡 Student ID Note Karo

Har student enroll hone pe auto-generated ID milti hai (STU-2025-XXX). Ise school register mein likho — student login ke liye zaroorat padegi.

🔒 Password Security

Owner password strong rakho — "Raunak@123" jaisa. Naam + symbol + number. Staff passwords bhi alag alag rakho — ek shared password mat banana.

🔒 GitHub Repository Private Rakho

Agar GitHub pe host kar rahe ho toh repository Private rakho — warna school_data.js mein likhe passwords publicly readable ho jaayenge.

📅 Session End Routine

Har academic year ke aakhir mein: Full export karo → Archive save karo → New session ke liye settings update karo (currentSession: "2026-27").

📊 Marks Entry Order

Pehle exam banao → Phir marks enter karo → Phir "Result Declared" mark karo. Isi order mein karo — warna students ko nahi dikhega result.

🌐 Offline Kaam Karta Hai

Internet nahi hai? Koi baat nahi. HTML files local folder se bhi chalti hain. Import/export file se hoga. GitHub push tab karo jab internet mile.

📱 Mobile Pe Bhi Chalega

Koi bhi modern mobile browser pe yeh ERP kaam karta hai — Chrome Android ya Safari iOS. Responsive design hai. Staff zaroorat pe mobile se bhi access kar sakte hain.

🏆 Golden Rule — Ek Baat Yaad Rakho

Har din kaam khatam karte waqt Export karo. Baki sab bhool jao — sirf yeh ek cheez yaad rakho. Export file = saara data. Drive pe rakho = kabhi data loss nahi. Simple.

Agar yeh ek aadat ban gayi, toh system kitni bhi dafaa crash ho, browser delete ho, phone khoya ho — aapka data hamesha safe rahega.
