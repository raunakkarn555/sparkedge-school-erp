📋 Students Portal — Complete Guide
How to enroll students, manage records, set up student logins, track fees, export/import data, and more. Read once, you're set for the whole year.
Who Can Access This Page?
Owner
Full access — enroll, edit, delete, activate/deactivate, view fee status, export/import, set up student login accounts.
Staff (students permission ON)
Can view and enroll students. Cannot delete. Fee details visible only if they also have fees permission.
🔑 Login: Same password as admin.html. Open students.html and enter it. No separate password needed. Passwords live in school_data.js and are also copied into §1 of this file's script.
How to Enroll a Student
1
Click + Enroll Student (top-right). A large form opens with 4 sections: Personal, Academic, Guardian, and Address.
2
Personal Info: Full name, Date of Birth, Gender, Blood Group, Category (General/OBC/SC/ST), Aadhaar (optional).
3
Academic: Class (1–12, Nursery, LKG, UKG), Section (A/B/C/D), Roll Number, Admission Number, Session, Admission Date. Roll number and Admission No. are manual — enter your school's numbering.
4
Guardian: Name, Relation (Father/Mother/Guardian), Phone, Email, Occupation. This is who receives fee receipts and notices.
5
Address: Full address, City, State, Pincode, Previous School (optional for transfer students).
6
Optional: Upload a passport-size photo (click the photo box, max 500 KB). Stored as base64 in the browser.
✓
Click Save Student. The ID is auto-generated as STU-YYYY-NNN (e.g. STU-2025-001). Note this ID — you'll need it to set up the student's login.
Setting Up a Student Login Account
For student-login.html (student self-service portal)
Student login accounts are separate from admin/staff accounts. Students use student-login.html with their Student ID + password. These are set up in school_data.js.
1
Enroll the student first (above). Note their auto-generated ID — shown on their profile card, format: STU-YYYY-NNN.
2
Open school_data.js in a code editor. Find the STUDENT_ACCOUNTS = [...] array (around line 75).
3
Add a block like this inside the [ ]:
{
  id:      'STU-2025-001',  // copy exact ID from profile
  password: 'Ananya@01',   // share with student/guardian
},
4
Save file → git add . → git commit -m "add student login" → git push → GitHub Pages updates in ~60 sec.
✓
Share with the student/guardian via WhatsApp: the URL (student-login.html), their Student ID, and their password. Password tip: use FirstName@RollNo (e.g. Ananya@15).
Editing, Transfer & Deactivation
✏
Edit: Click the pencil icon on a student row or on the profile card. All fields are editable. Save to update.
↗
Class Promotion / Transfer: Edit the student → change the Class, Section, Session fields → Save. Their old fee/marks records are preserved (linked by Student ID).
⏸
Deactivate (for TC / leaving students): Click the pause icon → student is marked inactive and greyed out. All records preserved. Reactivate with play icon if they re-enroll.
🗑
Delete: Permanent. Only available to Owner. Prefer Deactivate — deletion also loses any linked fee/marks history.
Export & Import — Data Sync Across Devices
💡 Data lives in each browser's localStorage. Export/Import moves it between devices — no cloud sync needed.
📤
Export: Sidebar or topbar → Export Data → downloads school-data-YYYY-MM-DD.json. Contains ALL data (students, teachers, fees, marks, docs).
📲
Share: Send the JSON file via WhatsApp, Google Drive, email, or USB to your colleague.
📥
Import: Open any portal → Import Data → upload file or paste JSON → Import & Merge. Existing records with the same ID are updated; new records are added; nothing is deleted.
✓
Best practice: Owner exports after every data-entry session. Share with team. Everyone imports at start of work. Simple, reliable, zero cost.
⚠️ Conflict: If two people edit the same student simultaneously and then import, the last import wins. Coordinate who edits what to avoid overwrites.
